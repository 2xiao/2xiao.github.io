# LeetCode题解

说到 LeetCode，作为一个程序员来说，应该不陌生，近几年参加面试都会提到它。国内外的程序员用它刷题主要是为了面试。

本书是LeetCode Cookbook系列的JS版本，系列发起者是[@halfrost](https://github.com/halfrost)，他的LeetCode-Go[项目](https://github.com/halfrost/LeetCode-Go)和[书籍](https://books.halfrost.com/leetcode/)，在他的鼓励下，我开始写JS版本。

一边刷题，一边记录一下做题心得，解题方法，对于自己来说，把解题思路和代码详细写下俩，让人能轻松理解，也是一种提高。

## 使用说明

- 本电子书的左上角有搜索栏，可以迅速帮你找到你想看的章节和题号。
- 关于题解，笔者建议这样使用：先自己读题，思考如何解题。如果 15 分钟还没有思路，那么先看笔者的解题思路，但是不要看代码。有思路以后自己用代码实现一遍。如果完全不会写，那就看笔者提供的代码，找出自己到底哪里不会写，找出问题记下来，这就是自己要弥补的知识漏洞。如果自己实现出来了，提交以后有错误，自己先 debug。AC 以后没有到 100% 也先自己思考如何优化。如果每道题自己都能优化到 100% 了，那么一段时间以后进步会很大。所以总的来说，实在没思路，看解题思路；实在优化不到 100%，看看代码。
